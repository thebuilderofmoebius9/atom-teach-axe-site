RPG Item Glossary

Item Glossary: ศัพท์เทคนิค

หน้านี้แปลคำอย่าง tmux, shell, PATH, webhook, broker, systemd ให้คนไม่เขียนโค้ดอ่านรู้เรื่องก่อนเข้า lesson หลัก

วิธีใช้: ถ้าเจอคำแปลกในเควส ให้เปิดหน้านี้แล้วอ่าน 3 ช่อง: แปลว่าอะไร · ทำไมต้องรู้ · ภาพจำ

tmux

สมุดจอหลายหน้าใน terminal

ทำไมต้องรู้
ใช้เปิดงานยาว ๆ แล้วกลับมาดูต่อได้ แม้หน้าต่างเดิมหลุด
ภาพจำ
เหมือนโต๊ะทำงานที่มีหลายแฟ้มเปิดค้างไว้ ไม่ต้องเริ่มใหม่ทุกครั้ง
ตัวอย่างในเว็บนี้
ใช้ดูว่า loop หรือ agent ยังทำงานอยู่ไหม

terminal / shell

หน้าต่างพิมพ์คำสั่งให้คอมพิวเตอร์ทำงาน

ทำไมต้องรู้
หลายระบบหลังบ้านไม่มีปุ่มสวย ๆ ต้องสั่งผ่านข้อความ
ภาพจำ
เหมือนช่องวิทยุสื่อสารกับเครื่องโดยตรง
ตัวอย่างในเว็บนี้
รัน build, test, deploy หรือดู log

CLI

โปรแกรมที่ใช้ผ่านคำสั่ง ไม่ใช่กดเมนู

ทำไมต้องรู้
ทำซ้ำได้ ตรวจได้ และเหมาะกับ automation
ภาพจำ
เหมือนสูตรอาหารที่เขียนเป็นบรรทัด ๆ ใครทำตามก็ได้ผลใกล้กัน
ตัวอย่างในเว็บนี้
git, curl, node, python, arra-cli

cwd

โฟลเดอร์ที่คำสั่งกำลังยืนอยู่ตอนนี้

ทำไมต้องรู้
คำสั่งเดียวกัน ถ้ายืนคนละที่ ผลอาจคนละเรื่อง
ภาพจำ
เหมือนส่งของจากคนละบ้าน เลขห้องเดียวกันแต่คนละตึก
ตัวอย่างในเว็บนี้
รัน build ใน repo ผิดที่แล้วหาไฟล์ไม่เจอ

PATH

รายชื่อที่คอมพิวเตอร์ใช้ค้นหาโปรแกรม

ทำไมต้องรู้
ถ้า PATH ผิด เครื่องจะบอกว่าไม่รู้จักคำสั่ง ทั้งที่โปรแกรมมีอยู่
ภาพจำ
เหมือนสมุดรายชื่อร้าน ถ้าไม่มีเบอร์ก็โทรหาไม่ได้
ตัวอย่างในเว็บนี้
agent รัน node/npm ไม่เจอเพราะ PATH ไม่เหมือน terminal คน

env / environment variable

ค่าตั้งต้นลับหรือค่าคอนฟิกที่ส่งให้โปรแกรมตอนรัน

ทำไมต้องรู้
ใช้แยก config จาก code และไม่ควรพิมพ์ secret ลงเว็บ
ภาพจำ
เหมือนซองคำสั่งเฉพาะงานที่แนบไปกับคนทำงาน
ตัวอย่างในเว็บนี้
API token, mode, database URL

secret / token

กุญแจดิจิทัลที่ใช้ยืนยันสิทธิ์

ทำไมต้องรู้
ถ้าหลุด คนอื่นอาจเข้าระบบแทนเราได้
ภาพจำ
เหมือนกุญแจบ้าน ไม่ใช่ป้ายชื่อบ้าน
ตัวอย่างในเว็บนี้
OAuth token, GitHub token, API key

credential placeholder

คำแทนค่า secret ในเอกสาร public

ทำไมต้องรู้
สอนขั้นตอนได้โดยไม่เผยกุญแจจริง
ภาพจำ
เหมือนเขียนว่า <รหัสผ่านของคุณ> แทนการเขียนรหัสจริง
ตัวอย่างในเว็บนี้
ใส่ชื่อ placeholder เช่น your-token-here แทนค่าจริง

SQLite

ฐานข้อมูลไฟล์เดียวที่พกง่าย

ทำไมต้องรู้
เหมาะกับ mirror/local index ที่ต้องค้นเร็วโดยไม่ตั้ง server ใหญ่
ภาพจำ
เหมือนสมุดบัญชีเล่มเดียวที่มีสารบัญค้นได้
ตัวอย่างในเว็บนี้
เก็บ Discord mirror เป็นไฟล์ .sqlite

FTS5

ระบบค้นคำใน SQLite

ทำไมต้องรู้
ตอบคำถามว่า ‘ข้อความไหนมีคำนี้’ ได้เร็ว
ภาพจำ
เหมือนช่องค้นหาในสมุดโน้ต
ตัวอย่างในเว็บนี้
หา message ที่พูดถึง PR หรือ tmux

TF-IDF

เครื่องหา ‘คำที่เด่นผิดปกติ’ ในช่วงหนึ่ง

ทำไมต้องรู้
ช่วยรู้ว่าช่วงนี้ห้องกำลังสนใจเรื่องอะไรโดยไม่ต้องอ่านทุกข้อความ
ภาพจำ
เหมือนปากกาไฮไลต์ที่วงคำสำคัญบนหน้าเอกสาร
ตัวอย่างในเว็บนี้
เทียบสัปดาห์นี้กับทั้งเดือนแล้วพบว่า tmux เด่น

backfill

ดึงข้อมูลย้อนหลังมาเติมให้ครบ

ทำไมต้องรู้
ถ้ามีแต่ข้อความใหม่หลังบอทออนไลน์ จะไม่เห็นอดีต
ภาพจำ
เหมือนขอย้อนดูสมุดเช็กชื่อก่อนวันที่เราเข้าห้อง
ตัวอย่างในเว็บนี้
ดึง Discord messages ตั้งแต่วันแรกของโรงเรียน

gateway

ประตูสดที่ส่งเหตุการณ์ใหม่เข้ามา

ทำไมต้องรู้
เห็นสิ่งที่เกิดหลังเชื่อมต่อ แต่ไม่ได้แปลว่าย้อนอดีตครบ
ภาพจำ
เหมือนนั่งเฝ้าประตู เห็นคนเข้าออกตั้งแต่ตอนเริ่มเฝ้า
ตัวอย่างในเว็บนี้
Discord gateway ส่ง message ใหม่ให้ bot

REST API

ทางถามข้อมูลย้อนหลังหรือสั่งงานผ่าน request

ทำไมต้องรู้
ใช้ดึงข้อมูลที่ gateway ไม่ได้ส่งมา หรือสั่งระบบแบบเจาะจง
ภาพจำ
เหมือนโทรไปถามสำนักงานทะเบียน ไม่ใช่นั่งเฝ้าประตู
ตัวอย่างในเว็บนี้
backfill message history จาก Discord

webhook

ประตูรับแจ้งเหตุจากระบบอื่น

ทำไมต้องรู้
ทำให้ระบบหนึ่งสะกิดอีกระบบได้อัตโนมัติ
ภาพจำ
เหมือนกริ่งหน้าบ้านที่ระบบอื่นมากดได้
ตัวอย่างในเว็บนี้
GitHub แจ้ง deploy หรือ bot ส่ง status เข้า control room

broker

คนกลางรับฝากและส่งต่อข้อความ

ทำไมต้องรู้
ช่วยให้หลายระบบคุยกันได้โดยไม่ต้องรู้จักกันตรง ๆ
ภาพจำ
เหมือนไปรษณีย์กลางของหมู่บ้าน
ตัวอย่างในเว็บนี้
message broker รับงานจาก agent แล้วส่งต่อ worker

queue

แถวคิวของงานที่รอทำ

ทำไมต้องรู้
กันระบบล้น และทำให้งานยาวไม่หายเมื่อมีหลายคำสั่ง
ภาพจำ
เหมือนบัตรคิวร้านอาหาร
ตัวอย่างในเว็บนี้
Discord สั่งงานยาวแล้ว worker ค่อยหยิบไปทำ

worker

ตัวทำงานที่หยิบงานจาก queue ไปทำจริง

ทำไมต้องรู้
แยกคนรับคำสั่งออกจากคนทำงานยาว ช่วยให้ระบบไม่ค้าง
ภาพจำ
เหมือนพนักงานหลังครัวที่รับออเดอร์จากหน้าร้าน
ตัวอย่างในเว็บนี้
queue worker สร้างเว็บหรือส่งไฟล์แนบ

systemd service

บริการหลังบ้านที่ Linux คอยเปิด/ดูแลให้

ทำไมต้องรู้
ทำให้ bot หรือเว็บทำงานต่อหลังเครื่องรีสตาร์ตได้
ภาพจำ
เหมือนยามอาคารที่คอยเปิดไฟและเช็กว่าร้านยังเปิดอยู่
ตัวอย่างในเว็บนี้
atom-native.service

log

บันทึกเหตุการณ์ที่ระบบทำไว้

ทำไมต้องรู้
ใช้พิสูจน์ว่าเกิดอะไรจริง ไม่ต้องเดา
ภาพจำ
เหมือนกล้องหน้ารถของระบบ
ตัวอย่างในเว็บนี้
ดู error ตอน deploy หรือดูว่าบอทส่งข้อความสำเร็จไหม

HTTP 200

เว็บตอบว่าเปิดได้ปกติ

ทำไมต้องรู้
เป็นหลักฐานขั้นต่ำว่า route นั้นยังไม่พัง
ภาพจำ
เหมือนเคาะประตูแล้วมีคนตอบ ‘อยู่ครับ’
ตัวอย่างในเว็บนี้
ตรวจหน้า /glossary/ ได้ 200

static site

เว็บที่เป็นไฟล์ HTML/CSS/JS ไม่มี server หลังบ้าน

ทำไมต้องรู้
เร็ว แชร์ง่าย และเหมาะกับบทเรียน public-safe
ภาพจำ
เหมือนโบรชัวร์ดิจิทัลที่เปิดอ่านได้ทุกที่
ตัวอย่างในเว็บนี้
GitHub Pages ของ Atom Teach Axe

GitHub Pages

บริการฟรีของ GitHub สำหรับเผยแพร่เว็บ static

ทำไมต้องรู้
ส่งลิงก์ให้คนอื่นเปิดได้ ไม่ต้องแนบไฟล์ HTML
ภาพจำ
เหมือนเอาโปสเตอร์ไปติดบอร์ดสาธารณะ
ตัวอย่างในเว็บนี้
thebuilderofmoebius9.github.io/atom-teach-axe-site

commit

จุดบันทึกการเปลี่ยนแปลงใน git

ทำไมต้องรู้
ย้อนดูได้ว่าแก้อะไร เมื่อไหร่ และใครแก้
ภาพจำ
เหมือนเซฟเกมก่อนเข้าด่านใหม่
ตัวอย่างในเว็บนี้
commit 645f443 เพิ่ม quiz 75 ข้อ

repo

กล่องเก็บโค้ดและประวัติการแก้ไข

ทำไมต้องรู้
ทำให้เว็บ/โปรเจกต์แชร์และตรวจย้อนหลังได้
ภาพจำ
เหมือนแฟ้มงานกลางที่มีประวัติทุกเวอร์ชัน
ตัวอย่างในเว็บนี้
atom-teach-axe-site repo

PR / Pull Request

คำขอรวมงานแก้ไขเข้าโปรเจกต์หลัก

ทำไมต้องรู้
ให้คน review ก่อนเปลี่ยนของจริง
ภาพจำ
เหมือนส่งร่างเอกสารให้หัวหน้าเซ็นก่อนประกาศ
ตัวอย่างในเว็บนี้
เสนอ patch ให้ repo upstream

build

ขั้นตอนแปลง source ให้พร้อมใช้งาน

ทำไมต้องรู้
เว็บหรือโปรแกรมบางอย่างต้องประกอบไฟล์ก่อนเผยแพร่
ภาพจำ
เหมือนประกอบชิ้นส่วน LEGO ให้เป็นรถก่อนโชว์
ตัวอย่างในเว็บนี้
scripts/build_site.py สร้าง HTML จาก data JSON

deploy

นำของที่ build แล้วขึ้นให้คนใช้งานจริง

ทำไมต้องรู้
งานยังไม่จบจนกว่าผู้ใช้เปิดได้จากลิงก์จริง
ภาพจำ
เหมือนย้ายของจากโต๊ะทำงานไปขึ้นเวที
ตัวอย่างในเว็บนี้
push ไป GitHub Pages แล้วตรวจ live URL

htmlhint / node --check

เครื่องมือตรวจว่าไฟล์เว็บหรือ JavaScript ไม่พังพื้นฐาน

ทำไมต้องรู้
จับ error ง่าย ๆ ก่อนส่งให้คนเปิด
ภาพจำ
เหมือนตรวจคำสะกดก่อนส่งรายงาน
ตัวอย่างในเว็บนี้
htmlhint ผ่าน 17 ไฟล์

kill switch

ปุ่มหยุดฉุกเฉินของ loop หรือระบบยาว

ทำไมต้องรู้
ถ้างานวิ่งผิดทาง ต้องหยุดได้เร็วและชัด
ภาพจำ
เหมือนปุ่มแดงบนเครื่องจักร
ตัวอย่างในเว็บนี้
หยุด agent loop ที่กำลังทำผิด scope

P2P

เครื่องหลายเครื่องคุยกันโดยไม่ต้องผ่านศูนย์กลางเดียวเสมอ

ทำไมต้องรู้
ช่วยกระจายงานและลดการพึ่งจุดเดียว
ภาพจำ
เหมือนเพื่อนส่งของต่อกันเอง ไม่ต้องผ่านสำนักงานใหญ่ทุกครั้ง
ตัวอย่างในเว็บนี้
agent/fleet sync ระหว่างเครื่อง

mesh

เครือข่ายที่หลายเครื่องเชื่อมกันเป็นตาข่าย

ทำไมต้องรู้
ทำให้ fleet ยืดหยุ่นและยังสื่อสารได้เมื่อบางเส้นทางเปลี่ยน
ภาพจำ
เหมือนถนนในเมืองที่มีหลายทางไปถึงจุดหมาย
ตัวอย่างในเว็บนี้
Netbird mesh สำหรับเครื่องหลายตัว

WASM

วิธีแพ็ก logic ให้ไปรันได้หลายที่แบบเบา ๆ

ทำไมต้องรู้
ช่วยให้โค้ดบางส่วนย้ายข้ามเครื่อง/อุปกรณ์ได้ง่ายขึ้น
ภาพจำ
เหมือนปลั๊กแปลงไฟสากลสำหรับโปรแกรมเล็ก ๆ
ตัวอย่างในเว็บนี้
ESP32-WASM desk pet

ESP32

บอร์ดคอมพิวเตอร์จิ๋วราคาถูกสำหรับทำอุปกรณ์จริง

ทำไมต้องรู้
ทำให้บทเรียน AI หลุดจากจอ ไปจับฮาร์ดแวร์ได้
ภาพจำ
เหมือนสมองเล็ก ๆ สำหรับของเล่นอัจฉริยะ
ตัวอย่างในเว็บนี้
desk pet, voice device, sensor

STT

แปลงเสียงพูดเป็นข้อความ

ทำไมต้องรู้
เป็นสะพานจาก voice ไปหา LLM
ภาพจำ
เหมือนเลขาที่ถอดเทปให้ AI อ่าน
ตัวอย่างในเว็บนี้
ผู้ใช้พูด → STT → LLM ตอบ

LLM

โมเดลภาษาขนาดใหญ่ที่เข้าใจและสร้างข้อความ

ทำไมต้องรู้
เป็นสมองภาษาของ Oracle แต่ไม่ใช่ระบบทั้งหมด
ภาพจำ
เหมือนนักคิดในห้องควบคุม ต้องมีเครื่องมือและหลักฐานช่วย
ตัวอย่างในเว็บนี้
Codex/ChatGPT ช่วยสรุปหรือเขียนโค้ด

OAuth

วิธีให้สิทธิ์แอปโดยไม่ต้องบอกรหัสผ่านตรง ๆ

ทำไมต้องรู้
ปลอดภัยกว่าเอารหัสจริงไปใส่ใน script
ภาพจำ
เหมือนบัตรผ่านชั่วคราวที่ออกโดยเจ้าของอาคาร
ตัวอย่างในเว็บนี้
login YouTube CLI เพื่ออัปโหลดคลิป

cron job

งานที่ตั้งเวลาให้รันเอง

ทำไมต้องรู้
ใช้ทำงานซ้ำ เช่น scan หรือสร้าง draft ทุกวัน โดยไม่ต้องกดเอง
ภาพจำ
เหมือนนาฬิกาปลุกที่ปลุกระบบมาทำงานตอนเช้า
ตัวอย่างในเว็บนี้
ตั้ง daily scan ให้หา lesson candidates แต่ยังไม่ auto-publish

Doctrine

หลักคำสอนหรือชุดข้อเท็จจริงเชิงคัมภีร์ที่บันทึกไว้

ทำไมต้องรู้
ช่วยให้เข้าใจขอบเขตความรู้เชิงตำราที่ AI สามารถเข้าถึงได้ทันทีโดยไม่ต้องรันเครื่องมือ
ภาพจำ
เหมือนสูตรอาหารในสมุดเก่าแก่: บอกขั้นตอนได้ แต่ตัวสมุดไม่ได้ชิมอาหารนั้นจริง ๆ
ตัวอย่างในเว็บนี้
การอธิบายความหมายของพระพุทธเจ้า หรือหลักธรรม อริยสัจ 4 ในหน้าประวัติศาสตร์

Provenance

แหล่งที่มาและประวัติหลักฐานของข้อมูล

ทำไมต้องรู้
ช่วยให้รู้ว่าข้อมูลที่อ้างอิงมาจากข้อความใด ห้องใด หรือเซนเซอร์ใด เพื่อความโปร่งใสและตรวจสอบซ้ำได้
ภาพจำ
เหมือนป้ายแปะหลังขวดยา: บอกว่าใครผลิต วันที่เท่าไหร่ และมาจากล็อตไหน
ตัวอย่างในเว็บนี้
การระบุ Message ID ของ Discord หรือ URL บล็อกสดในการอ้างอิงข้อมูลของเพื่อนร่วมทีม

Negative Probe

ยิง URL ปลอมที่ไม่มีอยู่จริงแล้วคาดหวังให้ได้ HTTP 404 เพื่อพิสูจน์ว่าเซิร์ฟเวอร์ route จริง

ทำไมต้องรู้
แยกเว็บที่ route หน้าจริงออกจากหน้า SPA/catch-all ที่คืน 200 ให้ทุก URL ซึ่งหลอกให้คิดว่าหน้าใหม่ขึ้น live
ภาพจำ
เหมือนกดเรียกห้องที่ไม่มีในตึก ถ้าระบบบอก 'ไม่มีห้องนี้' แสดงว่ามีระบบนำทางจริง ถ้าเปิดได้หมดแปลว่าเป็นประตูกล
ตัวอย่างในเว็บนี้
curl หน้าจริงได้ 200 และ curl /__fake_probe__/ ได้ 404 จึงยืนยันว่าเว็บ route จริง

Drift (pushed≠live)

ช่องว่างระหว่างโค้ดที่ push ขึ้น repo แล้ว กับสิ่งที่ผู้ใช้เห็นจริงบน live ซึ่งต้องรอ pipeline deploy

ทำไมต้องรู้
กันการอ้างว่า verified live ทั้งที่จริง ๆ ยังอ่านเวอร์ชันเก่า เพราะ GitHub Actions Pages ใช้เวลา 1-2 นาทีหลัง push
ภาพจำ
เหมือนส่งของ: ได้ใบเสร็จ (push) ไม่แปลว่าของถึงมือลูกค้า (live) ต้องรอและเช็คอีกที
ตัวอย่างในเว็บนี้
push commit แล้วรอ 1-2 นาที ค่อย curl สดเช็คว่าหน้าใหม่ขึ้นจริงก่อนรายงาน verified

Nonce Round-Trip

สร้างค่าสุ่มสดตอนตรวจ ส่งผ่านระบบจริง แล้วต้องอ่านค่าเดิมกลับจากปลายทาง เพื่อพิสูจน์ว่า loop ทำงานสดจริง

ทำไมต้องรู้
log/screenshot เป็นหลักฐานอดีตที่ปลอมได้ แต่ค่าสุ่มที่เพิ่งสร้างแล้ววิ่งครบ loop ไม่มีทางเตรียมล่วงหน้า จึงใช้ตรวจรับงาน integration ได้แน่นกว่า
ภาพจำ
เหมือนสั่งให้เพื่อนชูสามนิ้วสด ๆ ตอนวิดีโอคอล — ท่าที่เพิ่งสั่งอัดไว้ล่วงหน้าไม่ได้
ตัวอย่างในเว็บนี้
สร้างเลขจาก date +%s%N ส่งเข้า channel ที่ bot อ่าน แล้วตรวจว่า bot รายงานเลขเดียวกันกลับมา

Zero-Dependency Client

โปรแกรมที่คุยกับ protocol (เช่น WebSocket, MQTT) โดยเขียนการแกะ/ประกอบข้อมูลเองทั้งหมด ไม่พึ่ง library ภายนอก

ทำไมต้องรู้
บังคับให้เข้าใจ protocol จริงทุกชั้น ตรวจสอบพฤติกรรมได้เอง และตัดความเสี่ยง supply chain ของ dependency — แลกกับภาระดูแลโค้ดเอง
ภาพจำ
เหมือนทำอาหารจากวัตถุดิบสดเองแทนซื้อสำเร็จรูป: ช้ากว่าแต่รู้ทุกอย่างที่อยู่ในจาน
ตัวอย่างในเว็บนี้
Gateway reader ที่ต่อ Discord ด้วย raw WebSocket เอง หรือ MQTT client ที่แกะ varint/CONNECT/SUBACK บน socket ตรง ๆ

False Negative

ผลตรวจ/ผลค้นที่ตอบว่า "ไม่มี" ทั้งที่ของมีอยู่จริง เพราะวิธีค้นไม่ตรงกับข้อมูล

ทำไมต้องรู้
false negative อันตรายกว่า false positive ในงานรายงาน เพราะทำให้สรุปความว่างปลอมแล้วหยุดหา — ต้องกันด้วย cross-query
ภาพจำ
เหมือนตรวจหากุญแจด้วยการมองแค่ลิ้นชักเดียว แล้วประกาศว่ากุญแจหาย
ตัวอย่างในเว็บนี้
scan vault ด้วย keyword อังกฤษได้ 0 ไฟล์ แต่ grep คำไทยเจอ 20 ไฟล์

Cross-Query

การค้นเรื่องเดียวกันด้วยวิธี/ภาษา/เครื่องมือที่ต่างกันอย่างน้อย 2 แบบ ก่อนสรุปผล

ทำไมต้องรู้
ผลลบจากวิธีเดียวมีขอบเขตแค่ query นั้น การค้นข้ามวิธีทำให้ผลลบน่าเชื่อขึ้นและจับ false negative ได้
ภาพจำ
เหมือนถามหาคนในตึกทั้งชื่อจริงและชื่อเล่น — ถามชื่อเดียวแล้วไม่เจอ ยังสรุปไม่ได้ว่าเขาไม่อยู่
ตัวอย่างในเว็บนี้
ค้น keyword ไทย+อังกฤษ หรือใช้ scan tool คู่กับ grep ตรง ๆ แล้วเชื่อฝั่งที่เจอ
