Verification & Proof · 2026-07-23

กฎที่พึ่งความจำ คือกฎที่รอวันพลาด — ย้ายกฎเข้า guardrail

กฎสำคัญที่อาศัยแค่วินัยและความจำของคน จะพลาดในครั้งที่คนลืม ถ้าความผิดพลาดนั้นย้อนกลับไม่ได้ (secret หลุด, ลิงก์เสียถึงมือคนอื่น) ต้องย้ายกฎไปเป็นเครื่องมือที่บังคับเองอัตโนมัติ

Mission Brief

Mission Brief อ่านย่อ

บทนี้คือการแปลงวินัยส่วนบุคคลให้เป็นระบบป้องกันที่ตรวจสอบได้

ใจความคือกฎที่อาศัยความจำมนุษย์มีอัตราพลาดสะสมเสมอ องค์กรที่จริงจังกับความเสี่ยงแบบย้อนกลับไม่ได้ ต้องย้ายกฎเหล่านั้นจากหัวคนไปอยู่ในเครื่องมือที่บังคับใช้อัตโนมัติ

ทำไมต้องรู้: ความเสียหายแบบ secret รั่วหรือของเสียถึงมือลูกค้า เกิดจากคนที่รู้กฎดีแต่พลาดจังหวะเดียว การลงทุนกับ guardrail อัตโนมัติราคาถูกกว่าการแก้เหตุการณ์ที่ย้อนไม่ได้เสมอ

เลนส์ตัดสินใจ: เวลาทีมเสนอ "จะระวังให้มากขึ้น" หลังเหตุการณ์ ให้ถามกลับว่า "อะไรจะขวางไม่ให้เกิดซ้ำโดยไม่ต้องพึ่งใครจำ" ถ้าคำตอบยังเป็นวินัย แปลว่ายังไม่ได้แก้ที่ระบบ

จำสั้น ๆ: พลาดแล้วย้อนได้ → วินัยพอ พลาดแล้วย้อนไม่ได้ → ต้องเป็นเครื่องมือบังคับ และต้องทดสอบด้วยเคสปลอมว่ามันบล็อกจริง

Human first

สอนแบบมนุษย์

จำแบบนี้

กฎไหนที่ถ้าพลาดแล้วแก้ไม่ได้ อย่าฝากไว้กับความจำ ให้เขียนเป็นเครื่องมือที่ขวางไว้อัตโนมัติแทน

ทำไมต้องรู้

เคสจริงในห้องเรียน: เคย commit ไฟล์ backup ที่มี token จริงตอน repo ยัง public จนต้อง reset token และเคยประกาศลิงก์บทความก่อนเว็บ build เสร็จ ทำให้เพื่อนกดเจอ 404 — ทั้งสองเรื่องรู้กฎอยู่แล้วแต่ลืมในจังหวะเดียว

ภาพจำ

เหมือนประตูกันเด็กตกบันได — บ้านที่พึ่งการเตือนว่า "อย่าลืมปิดประตู" จะมีวันที่ลืม แต่ประตูที่ปิดเองอัตโนมัติไม่ต้องพึ่งใครจำ

Practice

ลองทำก่อนดูเฉลย

เลือกกฎที่เคยพลาดมา 1 ข้อ แล้วถามว่า "ถ้าพลาดซ้ำ ย้อนกลับได้ไหม" ถ้าไม่ได้ ลองเขียน check เล็ก ๆ ที่ขวางไว้ก่อนเกิดเหตุ แล้วทดสอบด้วยเคสปลอมว่ามันบล็อกจริง
