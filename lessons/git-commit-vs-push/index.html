Commit ≠ Push — เก็บของกับส่งของคนละขั้นตอน

git commit คือถ่ายรูปโค้ดเก็บไว้ในเครื่องตัวเอง ส่วน git push คือขั้นตอนที่ค่อยส่งรูปนั้นขึ้น GitHub ให้คนอื่นเห็น

Mission Brief

Mission Brief อ่านย่อ

บทนี้คือการแปลงหัวข้อ `Commit ≠ Push` ให้เป็นภาพใหญ่ที่ผู้บริหารตัดสินใจได้

ใจความคือ commit คือถ่ายรูปเก็บในเครื่อง ส่วน push คือส่งรูปนั้นออกไปให้โลกเห็น ประเด็นนี้ไม่ใช่แค่ syntax คำสั่ง แต่เป็นความเข้าใจว่าอะไรคือ "ปลอดภัยแล้ว" กับอะไรคือ "ยังเป็นความเสี่ยงเดี่ยว"

ทำไมต้องรู้: สำคัญเพราะทีมที่ไม่แยกสองคำนี้ออก จะเข้าใจผิดว่างานถูกสำรองไว้แล้วทั้งที่ยังอยู่ในเครื่องเดียว ถ้าเครื่องพัง งานที่ commit อย่างเดียวโดยไม่ push ก็หายได้เหมือนกัน

เลนส์ตัดสินใจ: มองผ่านเลนส์ผู้บริหาร: ก่อนบอกว่างาน "เสร็จแล้ว/ปลอดภัยแล้ว" ต้องเช็คว่า push ขึ้น remote จริงหรือยัง ไม่ใช่แค่ commit local

จำสั้น ๆ: commit = เก็บในเครื่อง, push = ส่งขึ้น GitHub ต้องทำสองขั้นตอนถึงจะปลอดภัยและแชร์ได้จริง

Human first

สอนแบบมนุษย์

จำแบบนี้

commit คือถ่ายรูปเก็บในเครื่อง ส่วน push คือส่งรูปนั้นออกไปให้โลกเห็น

ทำไมต้องรู้

ถ้าเข้าใจผิดว่า commit แล้ว = ขึ้น GitHub แล้ว จะเข้าใจผิดว่างานกู้คืนได้/แชร์ได้ทั้งที่ยังอยู่ในเครื่องตัวเองคนเดียว

ภาพจำ

commit เหมือนถ่ายรูปเก็บในมือถือตัวเอง ส่วน push เหมือนโพสต์รูปนั้นขึ้นโซเชียลให้คนอื่นเห็น — ถ่ายแล้วไม่ได้แปลว่าโพสต์แล้ว

Practice

ลองทำก่อนดูเฉลย
