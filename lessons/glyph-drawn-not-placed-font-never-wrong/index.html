วาด glyph ≠ วาง glyph — ฟอนต์ไม่เคยผิด วรรณยุกต์ที่หายไปไม่ได้หายที่ฟอนต์

ในการแสดงผลข้อความที่ข้อมูลไหลผ่านหลายชั้น (ฟอนต์ → ตัวจัดรูป → ตัววาดผล) ความถูกต้องคือสัญญาที่ต้องครบทุกชั้น ไม่ใช่คุณสมบัติของชั้นเดียว เมื่อวรรณยุกต์หรือสระบนหาย นิ้วมักชี้ไปที่ฟอนต์ ทั้งที่ฟอนต์มีทั้งรูป glyph และคำสั่งตำแหน่ง (anchor) ครบ — ตัวการจริงมักอยู่ที่ชั้นก่อนหน้าที่คืนความกว้างเป็น 0 ให้ตัวเสริม แล้วตัดทิ้งไปก่อนที่ฟอนต์จะได้วางมัน

Mission Brief

Mission Brief อ่านย่อ

อาการที่ปลายทางแสดงผล มักไม่ได้เกิดที่ปลายทาง อย่าโทษ component ที่เห็นชัดที่สุด

ในระบบที่ข้อมูลไหลผ่านหลายชั้น ความถูกต้องคือสัญญาที่ต้องครบทุกชั้น ไม่ใช่คุณสมบัติของชั้นเดียว เมื่อวรรณยุกต์หาย นิ้วมักชี้ที่ฟอนต์ ทั้งที่ฟอนต์มีทั้งรูปและคำสั่งตำแหน่งครบ ตัวการจริงอยู่ที่ชั้นก่อนหน้าที่คืนความกว้างเป็น 0 แล้วทิ้งตัวอักษรเสริมไปก่อน

ทำไมต้องรู้: เพราะการแก้ผิดชั้นเสียเวลาเป็นสิบเท่า วนปรับฟอนต์ ขนาด ถ่ายเทียบ แต่ไม่มีวันหาย เพราะข้อมูลตายตั้งแต่ก่อนถึงฟอนต์ การไล่จาก ข้อมูลหายตอนไหน แทน ใครผิด ถึงจะเจอ root cause

เลนส์ตัดสินใจ: เวลาเจออาการที่ component หนึ่ง ให้ถามสองชั้น: component นี้ได้รับข้อมูลครบหรือไม่ และมีชั้นไหนก่อนหน้าที่อาจตีความข้อมูล zero-width ว่าทิ้งได้ คำตอบมักอยู่ที่ชั้นก่อนหน้า ไม่ใช่ที่ตัวที่ถูกโทษ

จำสั้น ๆ: วาด glyph ≠ วาง glyph ฟอนต์ไม่เคยผิด ให้หาจุดที่ข้อมูลหายระหว่างทาง ไม่ใช่จุดที่โดนโทษ

จำแบบนี้

การวาดรูปตัวอักษร (glyph) เก็บไว้ในฟอนต์ กับ การเอารูปนั้นไปวางให้ถูกที่บนจอ คือคนละเรื่อง ฟอนต์ไทยที่ดีมีทั้งรูปวรรณยุกต์ครบ และมีคำสั่งตำแหน่งที่บอกว่าให้วางไม้เอก ไม้โท หรือสระบนตรงไหน แต่ถ้าโปรแกรมที่อ่านฟอนต์ไม่เปิดอ่านคำสั่งตำแหน่งเลย หรือทิ้งตัวอักษรที่ไม่มีความกว้างของตัวเองไปก่อนแล้ว วรรณยุกต์ก็หาย ทั้งที่ฟอนต์ไม่ได้ผิด

ทำไมต้องรู้

เคสจริงในห้องเรียน: มีหนังสือเล่มหนึ่งไล่ debug ว่าทำไม Linux console อ่านภาษาไทยไม่ได้ สระบนและวรรณยุกต์หายหมด ผู้ดูแลวนแก้ฟอนต์ ปรับขนาด ถ่ายรูปเทียบหลายสิบรอบ กว่าจะลงไปอ่าน source code แล้วเจอว่าตัวที่ทิ้งวรรณยุกต์คือฟังก์ชันคำนวณความกว้างที่คืน 0 ให้ตัวเสริม แล้วตัดทิ้งตั้งแต่ยังไม่ถึงมือตัววาดผล ปัญหาไม่ได้อยู่ที่ฟอนต์เลย

ภาพจำ

เหมือนไปรษณีย์ที่โทษคนออกแบบแสตมป์ว่าวาดที่อยู่ปลายทางไม่ชัด ทั้งที่จริง ๆ จดหมายหายเพราะพนักงานคัดแยกฉีกซองเล็ก ๆ ทิ้งก่อนส่งต่อ ไม่ใช่เพราะแสตมป์ผิด
