Pushed ≠ Live — ยืนยันเว็บจริงต้องตรวจสด ไม่ใช่แค่ git push

git push สำเร็จไม่ได้แปลว่าหน้าเว็บขึ้น live แล้ว เพราะขั้นตอน deploy (เช่น GitHub Actions Pages) มีช่วง drift 1-2 นาที การยืนยันว่าเว็บใช้งานจริงต้องยิง HTTP สด: หน้าจริงต้องได้ 200 และ URL ปลอมต้องได้ 404 เพื่อพิสูจน์ว่าเซิร์ฟเวอร์ route จริง ไม่ใช่หน้า SPA ที่คืน 200 ให้ทุก URL

Mission Brief

Mission Brief อ่านย่อ

แยก 'push สำเร็จ' ออกจาก 'live จริง' ด้วย HTTP probe คู่ positive/negative เพื่อกันรายงานเท็จ

git push สำเร็จเป็นเพียงสถานะฝั่ง repo ส่วนสถานะที่ผู้ใช้เห็นจริงต้องรอ deploy ผ่าน pipeline ซึ่งมี drift การยืนยัน live ที่ซื่อตรงต้องใช้ HTTP probe สดสองด้าน: หน้าจริงต้อง 200 และหน้าปลอมต้อง 404 เพื่อแยกเว็บที่ route จริงออกจากหน้า catch-all ที่คืน 200 หมด

ทำไมต้องรู้: สำคัญเพราะห้องเรียนเพิ่งเจอเคส 'pushed แล้วอ้าง verified 200 แต่จริง ๆ ยัง drift' ทำให้เพื่อนคลิกตามแล้วเจอเวอร์ชันเก่าหรือหน้าพัง จึงต้องมีกติกา verify สดทุกครั้งก่อนรายงาน

เลนส์ตัดสินใจ: มองผ่านเลนส์ผู้บริหาร: อย่าให้ทีมรายงานสถานะจากสิ่งที่ตั้งใจทำ (push) แต่ให้รายงานจากผลลัพธ์ที่วัดได้จริง (HTTP สด + negative probe) เพื่อให้ตัวเลขและสถานะที่ส่งต่อเชื่อถือได้

จำสั้น ๆ: push สำเร็จ ≠ live ต้อง curl สดคู่กัน หน้าจริง 200 และหน้าปลอม 404 ถึงจะเรียกว่า verified live

Human first

สอนแบบมนุษย์

จำแบบนี้

แค่ push โค้ดขึ้น repo ยังไม่พอจะบอกว่าเว็บ live แล้ว เพราะระบบ deploy อย่าง GitHub Pages ใช้เวลา 1-2 นาทีกว่าจะอัปเดตจริง และบางครั้งเว็บดูเหมือนผ่านเพราะคืน 200 ทุก URL ทั้งที่จริง ๆ เป็นหน้า SPA ก้อนเดียวที่ catch หมด วิธีตรวจให้แน่ใจคือยิง HTTP สดทั้งหน้าจริง (ต้อง 200) และ URL ปลอม (ต้อง 404)

ทำไมต้องรู้

เพื่อกันคำกล่าวอ้างว่า 'verified live' ทั้งที่จริง ๆ ยังอ่านเวอร์ชันเก่าหรือไม่ได้ route จริง จนทำให้รายงานส่งไปแล้วเพื่อนคลิกตามแล้วเจอหน้าพัง

ภาพจำ

เหมือนส่งพัสดุ: ได้ใบเสร็จรับของ (git push success) ยังไม่แปลว่าของถึงมือลูกค้า (live) ต้องโทรไปเช็คเบอร์จริงว่าได้รับหรือยัง และลองส่งผิดเบอร์ดูว่าระบบปฏิเสธจริงไหม

ลองทำก่อนดูเฉลย

หลัง push รอ 1-2 นาทีแล้ว curl สด 2 ครั้ง: URL จริงของบทเรียนใหม่ (คาดหวัง 200) กับ URL ปลอม เช่น /__fake_probe__/ (คาดหวัง 404) ถ้าทั้งคู่ตรงจึงรายงานว่า verified live
