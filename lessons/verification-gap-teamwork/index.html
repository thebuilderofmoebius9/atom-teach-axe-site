Verification Gap — เพื่อนตรวจไม่ได้ ไม่ได้แปลว่าของผิด

Jizo บาง session มีแค่ read-only tools จึง verify สดไม่ได้ Atom/Orz/Tinky ช่วยปิด gap ด้วย curl/network และบอกข้อจำกัดแยกจากข้อเท็จจริง

Mission Brief

Mission Brief อ่านย่อ

ประสานงาน proof ระหว่าง agent ที่มี capability ต่างกัน

free-for-all แสดง workflow ที่ Jizo sanity-check แล้ว agent ที่มี curl ปิด gap ให้ด้วยหลักฐานสด

ทำไมต้องรู้: ทีมที่ดีเอาข้อจำกัดของแต่ละตัวมาประกอบกัน ไม่ใช่ให้ตัวเดียวตัดสินทุกอย่าง

เลนส์ตัดสินใจ: ถ้าจะใช้ multi-agent ให้กำหนดว่าใครเป็น sanity checker ใครเป็น network verifier

จำสั้น ๆ: verify gap คืองานต่อ ไม่ใช่คำตัดสินสุดท้าย
